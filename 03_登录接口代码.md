# 第一版登录接口代码指南

在这个阶段，你需要在你的 Next.js 项目里写一点点“后端代码”。别慌，只需要复制粘贴即可！

## 先说最重要的一句

如果 Supabase 弹出：

- `Run without RLS`
- `Run and enable RLS`

你应该优先选择：

- `Run and enable RLS`

原因很简单：

- 更安全
- 更符合正式项目
- 不容易把数据库直接暴露出去

但是注意：

- 开启 RLS 之后，后端登录接口不要再用匿名 key 去查账号表
- 应该改成只在服务端使用 `service role key`

下面这份代码我已经按这个更安全的方式写好了。

---

## 第一步：创建服务端专用的 Supabase 连接文件

1. 在你的项目文件夹（`partnerx-app`）里，找到 `src` 文件夹。
2. 在 `src` 里面新建一个文件夹，叫 `lib`。
3. 在 `lib` 里面新建一个文件，叫 `supabase-server.ts`。
4. 把下面的代码**全部复制**进去保存：

```typescript
import { createClient } from '@supabase/supabase-js'

// 这是服务端专用连接，只能在后端代码里使用
const supabaseUrl = process.env.NEXT_PUBLIC_SUPABASE_URL!
const supabaseServiceRoleKey = process.env.SUPABASE_SERVICE_ROLE_KEY!

export const supabaseServer = createClient(
  supabaseUrl,
  supabaseServiceRoleKey,
  {
    auth: {
      autoRefreshToken: false,
      persistSession: false
    }
  }
)
```

这个文件只给后端接口用，不给前端页面直接用。

## 第二步：创建真正的登录 API 接口

1. 在你的项目 `src/app` 文件夹里，新建一个文件夹叫 `api`。
2. 在 `api` 里面再新建一个文件夹叫 `login`。
3. 在 `login` 里面新建一个文件叫 `route.ts`。（注意名字必须是这个）
   路径应该是这样的：`src/app/api/login/route.ts`
4. 把下面的代码**全部复制**进去保存：

```typescript
import { NextResponse } from 'next/server'
import { supabaseServer } from '@/lib/supabase-server'

export async function POST(request: Request) {
  try {
    // 1. 获取前端传过来的账号和密码
    const { username, password } = await request.json()

    if (!username || !password) {
      return NextResponse.json({ error: '账号和密码不能为空' }, { status: 400 })
    }

    // 2. 去数据库的 company_accounts 表里找这个账号
    const { data: user, error } = await supabaseServer
      .from('company_accounts')
      .select('id, company_id, role, username, password_hash, name, status')
      .eq('username', username)
      .single()

    // 3. 如果没找到账号，或者报错了
    if (error || !user) {
      return NextResponse.json({ error: '账号不存在或密码错误' }, { status: 401 })
    }

    // 4. 检查账号是否被禁用了
    if (user.status !== 'active') {
      return NextResponse.json({ error: '该账号已被停用，请联系管理员' }, { status: 403 })
    }

    // 5. 校验密码
    // 注意：第一版我们为了快点跑通，测试数据里直接存了 '123456' 明文。
    // 正式上线前，必须改成加密校验！比如使用 bcrypt 库。
    if (user.password_hash !== password) {
      return NextResponse.json({ error: '账号不存在或密码错误' }, { status: 401 })
    }

    // 6. 密码对了！把用户信息返回给前端
    // 注意不要把密码返回去！
    return NextResponse.json({
      message: '登录成功',
      user: {
        id: user.id,
        company_id: user.company_id,
        role: user.role,
        username: user.username,
        name: user.name
      }
    })

  } catch (err) {
    return NextResponse.json({ error: '服务器出了点小问题，请稍后再试' }, { status: 500 })
  }
}
```

好了！你的后端登录接口已经写完了！
接下来你只需要在你的前端页面里，当用户点击“登录”按钮时，向 `/api/login` 发送请求就可以了！

## 你还要补一个环境变量

除了之前的：

```env
NEXT_PUBLIC_SUPABASE_URL=你的URL
NEXT_PUBLIC_SUPABASE_ANON_KEY=[REDACTED]
```

你现在还要在 `.env.local` 里再补一行：

```env
SUPABASE_SERVICE_ROLE_KEY=[REDACTED]
```

你去 Supabase 后台这里找：

1. 左下角 `Project Settings`
2. 点击 `API`
3. 找到 `service_role`

注意：

- `service_role key` 非常重要
- 只能放在后端
- 不能写到前端页面代码里
